Introduzione

ALGORITMI: insieme ordinato e finito di pasi eseguibili e non ambigui che definiscono un procedimento che termina.

Progettazione:

  • Correttezza (Risolvere quello per cui è stato pensato).

  • Efficienza -> Analisi delle Risorse (Tempo, Spazio, Rete, Processore, Energia).

Programma: Insieme ordinato e finito di istruzioni scritte secondo le regole di uno specifico linguaggio di programmazione.

Algoritmo moltiplicativo alla Russa

L'algoritmo si basa sulla moltiplicazione del moltiplicando per 2 e la divisione del moltiplicatore per 2. Ogni volta che il moltiplicatore è dispari si prende il moltiplicando. La somma di quest'ultimi è la soluzione della moltiplicazione.

Criteri di costo uniforme

Ogni istruzione elementare utilizza un unità di tempo.
Ogni variabile elementare utilizza un unità di spazio indipendte.

Criterio di costo logaritmico: Proporzionale alla lunghezza dei valori coinvolti.

Complessità di Algoritmi

La complessità in tempo viene definita come funzione T : N --> N della lunghezza dell'input.

  • Caso peggiore
  • Caso migliore
  • Caso medio

Complessità rispetto al tempo:

Trovare un algoritmo A che risolve un problema P in tempo O(T(n))
        tale che il tempo sia sufficiente per risolvere P.
Dimostrare che ogni altro algoritmo che risolve P utilizza tempo Ω(T(n)).

Comlpessità rispetto allo spazio:

P ha complessità O(f(n)) rispetto a R quando esiste un algoritmo che risolve P il cui costo di esecuzione è O(f(n)).
P ha complessità O(g(n)) rispetto a R quando ogni algoritmo che risolve R ha complessità Ω(g(n)).


Algoritmo QuickSort

Procedimento

Caso base: soluzione immediata (Array ordinati)
Altrimenti:
                  • dividi l'Array in 2 parti.
                  • ordina le 2 parti separatamente.
                  • combina le soluzioni. 

ALGORITMO quickSort(Array A) 
 

      IF lunghezza di A > 1 THEN 

            scegli un elemento in A 

            B <-- {y appartiene a A t.c. y <= x} 

            C <-- {y appartiene a A t.c. y > x} 

            quickSort(B) //Ricorsione 

            quickSort(C) 

            A <-- concatenazione di B e di C

ALBERI BINARI

Gli alberi binari sono una struttura dati che può comprendere un albero vuoto oppure una radice e 2 sottalberi (che a loro volta saranno degli alberi o vuoti o composti da una radice e rispettivi sottoalberi).

Alberi binari:
•     Caso base: Albero vuoto.
•      Nodo con associati 2 alberi binari detti sottoalbero sinistro e sottoalbero destro.

Il minimo numero di nodi di un albero h è: h+1. (Degenera in lista)
Il massimo numero di nodi di un albero h è: 2 h+1 - 1 . (Albero completo)

log2 (n+1) -1  <=  h  <=  n-1


Vista in un Albero Binario

Strategia generale:

  • visito la radice
  • ogni volta che visoto un nodo memorizzo in un insieme S i suoi figli. Successivamente prelevo da S.

Vista in ampiezza (Coda)

ALGORITMO vistaAmpiezza( AlberoBinario r ) 


    C « coda vuota 

    C.equeue(r) 

    WHILE NOT( C.isEmpty() ) DO 

        n « c.dequeue() 

        IF n != null THEN 

            vista nodo associato a n 

            c.enqueue(n.sx) 

            c.enqueue(n.dx)

Vista in Profondità Anticipata ricorsiva 

ALGORITMO visitaInProfondità( AlberoBinario r ) 


    IF r != null THEN 

        visita la radice 

        visitaInProfondità(n.sx) 

        visitaInProfondità(n.dx)

Vista in Profondità Simmetrica ricorsiva 

ALGORITMO visitaInProfondità( AlberoBinario r ) 


    IF r != null THEN 

        visitaInProfondità(n.sx) 

        visita la radice 

        visitaInProfondità(n.dx)

Vista in Profondità Posticipata ricorsiva 

ALGORITMO visitaInProfondità( AlberoBinario r ) 


    IF r != null THEN 

        visitaInProfondità(n.sx) 

        visitaInProfondità(n.dx) 

        visita la radice

Sono le viste usate nelle espressioni matematiche. Esse usano la Notazione Polacca Inversa.


Alberi Binari di Ricerca

Preso un nodo, il suo sottoalbero sinstro è composta da tutte le chiavi minori del node, mentre il sottoalbero destro è composto da tutte le chiavi che sono maggiori o uguali al nodo.


Ricerca ricorsiva

Funzione trova( AlberoRicerca r, chiave k ) --> Nodo  


    IF r = null THEN 

        RETURN null 

    ELSE IF k < r.chiave THEN 

        RETURN trova( r.sx, k ) 

    ELSE IF k > r.chiave THEN 

        RETURN trova( r.dx, k ) 

    ELSE RETURN r

Ricerca interattiva

Funzione trova( AlberoRicerca r, chiave k ) --> Nodo  


    n « r 

    WHILE n != null AND n.chiave != k DO 

        IF k < n.chiave THEN 

            n « n.sx 

        ELSE 

            n « n.dx 

    ELSE RETURN n

N° di confronti nel caso peggiore: AltezzaAlbero + 1.


Inserimento Ricorsivo

Funzione inserimento( AlberoRicerca r, elemento d )
          --> AlberoRicerca 


    k « d.chiave 

    IF r = null THEN 

        r « riferimento a nuovo nodo
        r.chiave « k
        r.atricampi « d.altricampi
        r.sx « null
        r.dx « null 

    ELSE IF k < r.chiave THEN 

        r.sx « inserisci( r.sx, d ) 

    ELSE 

        r.dx « inserisci( r.dx, d ) 

    RETURN r

Inserimento Iterativo

Funzione inserimento( AlberoRicerca r, elemento d )
          --> AlberoRicerca 


    k « d.chiave 

    t « riferimento a nuovo nodo
    t.chiave « k
    t.atricampi « d.altricampi
    t.sx « null
    t.dx « null 

    padre « null
    n « r 

    WHILE n != null DO 

        padre « n 

        IF k < n.chiave THEN n « n.sx 

        ELSE n « n.dx 

    IF padre = null THEN r « t 

    ELSE IF k < padre.chiave THEN padre.sx « t 

    ELSE  padre.dx « t 

    RETURN r

N° di confronti: AltezzaAlbero.


Alberi Binari Perfettamente Bilanciati

Un albero binario è detto perfettamente bilanciato quando per ogni nodo, la differenza in valore assoluto dei nodi presenti nei suoi due sottoalberi è al più 1.



Alberi blanciati in altezza - AVL

Un albero binario è detto AVL quando per ogni nodo, la differenza in valore assoluto del altezza dei suoi due sottoalberi è al più 1.

N° di nodi di un albero AVL:
•  Masimo: 2h+1 -1 Θ(lg n)
•  Minimo: Θ(lg n)

Altezza logaritmica  -->  Ricerca in tempo logaritmico.

Ricerca, inserimento e cancellazione = Θ(lg n)
