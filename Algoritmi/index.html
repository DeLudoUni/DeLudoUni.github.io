Algoritmo QuickSort

Procedimento

Caso base: soluzione immediata (Array ordinati)
Altrimenti:
                  • dividi l'Array in 2 parti.
                  • ordina le 2 parti separatamente.
                  • combina le soluzioni. 

ALGORITMO quickSort(Array A) 
 

      IF lunghezza di A > 1 THEN 

            scegli un elemento in A 

            B <-- {y appartiene a A t.c. y <= x} 

            C <-- {y appartiene a A t.c. y > x} 

            quickSort(B) //Ricorsione 

            quickSort(C) 

            A <-- concatenazione di B e di C

ALBERI BINARI

Gli alberi binari sono una struttura dati che può comprendere un albero vuoto oppure una radice e 2 sottalberi (che a loro volta saranno degli alberi o vuoti o composti da una radice e rispettivi sottoalberi).

Alberi binari:
•     Caso base: Albero vuoto.
•      Nodo con associati 2 alberi binari detti sottoalbero sinistro e sottoalbero destro.

Il minimo numero di nodi di un albero h è: h+1. (Degenera in lista)
Il massimo numero di nodi di un albero h è: 2 h+1 - 1 . (Albero completo)

log2 (n+1) -1  <=  h  <=  n-1


Vista in un Albero Binario

Strategia generale:

  • visito la radice
  • ogni volta che visoto un nodo memorizzo in un insieme S i suoi figli. Successivamente prelevo da S.

Vista in ampiezza (Coda)

ALGORITMO vistaAmpiezza( AlberoBinario r ) 


    C « coda vuota 

    C.equeue(r) 

    WHILE NOT( C.isEmpty() ) DO 

        n « c.dequeue() 

        IF n != null THEN 

            vista nodo associato a n 

            c.enqueue(n.sx) 

            c.enqueue(n.dx)

Vista in Profondità Anticipata ricorsiva 

ALGORITMO visitaInProfondità( AlberoBinario r ) 


    IF r != null THEN 

        visita la radice 

        visitaInProfondità(n.sx) 

        visitaInProfondità(n.dx)

Vista in Profondità Simmetrica ricorsiva 

ALGORITMO visitaInProfondità( AlberoBinario r ) 


    IF r != null THEN 

        visitaInProfondità(n.sx) 

        visita la radice 

        visitaInProfondità(n.dx)

Vista in Profondità Posticipata ricorsiva 

ALGORITMO visitaInProfondità( AlberoBinario r ) 


    IF r != null THEN 

        visitaInProfondità(n.sx) 

        visitaInProfondità(n.dx) 

        visita la radice

Alberi Binari di Ricerca

Preso un nodo, il suo sottoalbero sinstro è composta da tutte le chiavi minori del node, mentre il sottoalbero destro è composto da tutte le chiavi che sono maggiori o uguali al nodo.


Ricerca ricorsiva

Funzione trova( AlberoRicerca r, chiave k ) --> Nodo  


    IF r = null THEN 

        RETURN null 

    ELSE IF k < r.chiave THEN 

        RETURN trova( r.sx, k ) 

    ELSE IF k > r.chiave THEN 

        RETURN trova( r.dx, k ) 

    ELSE RETURN r

Ricerca interattiva

Funzione trova( AlberoRicerca r, chiave k ) --> Nodo  


    n « r 

    WHILE n != null AND n.chiave != k DO 

        IF k < n.chiave THEN 

            n « n.sx 

        ELSE 

            n « n.dx 

    ELSE RETURN n

N° di confronti nel caso peggiore: AltezzaAlbero + 1.


Inserimento Ricorsivo

Funzione inserimento( AlberoRicerca r, elemento d )
          --> AlberoRicerca 


    k « d.chiave 

    IF r = null THEN 

        r « riferimento a nuovo nodo
        r.chiave « k
        r.atricampi « d.altricampi
        r.sx « null
        r.dx « null 

    ELSE IF k < r.chiave THEN 

        r.sx « inserisci( r.sx, d ) 

    ELSE 

        r.dx « inserisci( r.dx, d ) 

    RETURN r

Inserimento Iterativo

Funzione inserimento( AlberoRicerca r, elemento d )
          --> AlberoRicerca 


    k « d.chiave 

    t « riferimento a nuovo nodo
    t.chiave « k
    t.atricampi « d.altricampi
    t.sx « null
    t.dx « null 

    padre « null
    n « r 

    WHILE n != null DO 

        padre « n 

        IF k < n.chiave THEN n « n.sx 

        ELSE n « n.dx 

    IF padre = null THEN r « t 

    ELSE IF k < padre.chiave THEN padre.sx « t 

    ELSE  padre.dx « t 

    RETURN r

N° di confronti: AltezzaAlbero.
